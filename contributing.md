# Contribuindo com o Vault Bank API

## Branches
- `main` → código estável
- `develop` → desenvolvimento ativo
- `feature/<nome>` → novas funcionalidades

## Commits (convenção)
- `feat:` nova funcionalidade
- `fix:` correções
- `chore:` tarefas internas
- `refactor:` melhorias internas
- `docs:` documentação

## Pull Requests
- Descrever claramente o que foi alterado
- Incluir testes quando aplicável
- Não enviar PR diretamente para `main`
